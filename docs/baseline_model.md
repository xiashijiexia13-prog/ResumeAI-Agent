# 第一个基线模型：TF-IDF + Logistic Regression

## 任务定义

输入一对英文文本：简历与岗位描述；输出三档匹配等级：`No Fit`、`Potential Fit`、`Good Fit`。

## 数据划分

- 处理后训练数据：5,004 条。
- 从训练数据以 `random_state=42`、`stratify=label` 划出 80% 训练集（4,003 条）和 20% 验证集（1,001 条）。
- 官方验证数据经过清洗后保留为最终测试集（1,270 条）；它不参与参数选择。

## 模型设计

1. 将简历和岗位描述拼接为一个带字段前缀的文本。
2. 只在内部训练集拟合 TF-IDF：1–2 gram、`min_df=2`、最多 50,000 个特征。
3. 使用带类别平衡权重的 Logistic Regression 预测三档标签。

## 最终结果

| 数据集 | Accuracy | Macro-F1 | Weighted-F1 |
| --- | ---: | ---: | ---: |
| 内部验证集 | 0.5864 | 0.5881 | 0.5825 |
| 最终测试集 | 0.6181 | 0.6202 | 0.6149 |

最终测试集各类别 F1：`No Fit` 0.5998、`Potential Fit` 0.6028、`Good Fit` 0.6579。

## 解释

该模型能建立可复现且可解释的分类基线，但 TF-IDF 主要依赖词面重合，不能充分理解“同义技能”“可迁移经验”等语义关系。下一阶段将以同一最终测试集比较 Sentence Transformer。
